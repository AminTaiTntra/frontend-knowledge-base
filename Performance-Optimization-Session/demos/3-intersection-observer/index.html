📚 What is Intersection Observer?

The Intersection Observer API provides a way to asynchronously observe changes in the intersection of a target element with an ancestor element or with a top-level document's viewport. It's designed to replace expensive scroll event listeners for viewport-based functionality.

❌ Traditional Scroll Events

Example: 
window.addEventListener('scroll', () => {
  // Fires on EVERY scroll pixel!
  elements.forEach(el => {
    const rect = el.getBoundingClientRect();
    if (rect.top < window.innerHeight) {
      loadImage(el); // Heavy operation
    }
  });
});
Problems:
  • Fires continuously during scroll
  • Requires manual viewport calculations
  • Blocks main thread execution
  • High CPU usage and battery drain

✅ Intersection Observer

Example: 
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      loadImage(entry.target); // Efficient!
      observer.unobserve(entry.target);
    }
  });
});

elements.forEach(el => observer.observe(el));
Benefits:
  • Asynchronous, non-blocking operation
  • Browser-optimized calculations
  • Fires only when intersection changes
  • Minimal CPU usage and battery friendly

🎯 Common Use Cases

Lazy Loading Images: Load images only when they enter the viewport
Infinite Scrolling: Load more content as user approaches the bottom
Animation Triggers: Start animations when elements become visible
Analytics Tracking: Track which content users actually see

❌ Scroll Events Problems

  • Smooth, lag-free user experience
